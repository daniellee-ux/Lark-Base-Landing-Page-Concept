這是一段動態高度的 hero 內容。不同頁面可能有不同長度的文案——有時只是一句標語，有時是一整段產品敘述。

不管多長，cover 效果都要正常運作：讓用戶看完整段內容，白色區才從下往上蓋。

關鍵在 .dark { position: sticky; bottom: 0 } —— 底部定錨讓 dark 自然滾完所有內容，底部剛好接到視窗底時才釘住。

第二個關鍵是 .stage { padding-bottom: 100vh } —— 提供 cover 動畫所需的 100vh 滾動空間，不需要計算 dark 多高。

min-height: 100vh 是保險：內容少於一個視窗時也撐到 100vh，避免 sticky 在小元素上出怪事。

最後 .light { margin-top: -100vh } 本來就是常數，跟 dark 完全脫鉤。

隨便加內容都可以：多加一段，dark 就變高 100px；少一段，變矮 100px；cover 觸發點自動跟著內容高度移動。

把 hero 當成一整頁產品故事來寫：開場白、痛點、解法、關鍵數據、客戶引述，隨便多少段落都可以。只要 ResizeObserver 能偵測到 dark 高度變化，CSS 變數就會跟著更新。

假設產品經理今天臨時想多塞三段介紹文案——不用改 CSS、不用算 stage 高度、不用重新推導 light 的 margin-top。

或者想再加三段 FAQ、客戶 logo 列、甚至嵌入一段互動 demo，都不影響 cover 觸發時機。觸發點永遠是「dark 滾到底部接到視窗底」。

這就是把「變化的部分」（dark 實際高度）餵給 CSS 之後的彈性——三個零件（pin / overlap / stack）都保持單一公式，只有一個變數隨內容浮動。

把這頁當成壓力測試：dark 現在比第一版多約 600px。重新驗證時，sticky 觸發點、cover 動畫起始、完成點應該全部自動對齊新的高度。
